Serverless AI Platform

Katalyst – Product Data Platform

The Engineering Challenge:

Building a scalable scraper that could bypass enterprise-grade bot protections (Cloudflare/PerimeterX) without the operational nightmare and cost of managing thousands of headless browser instances (Puppeteer).

My Architecture:

I designed a Serverless "Waterfall" Scraping Pool on Firebase. The system attempts cheap native fetches first, cascades to ScrapFly, and falls back to Bright Data Unlocker only when necessary. Data is processed via Genkit flows that utilize RAG to inject historical successes into the prompt context.

The Tech Stack:

Next.js 14 • TypeScript • Google Genkit • Firebase App Hosting • Cloud Tasks • Firestore

Reduced scraping costs by 60% vs. headless browsers while maintaining 99.9% success rates on blocked sites.
Backend Performance & Revenue

Recommendation Engine Re-Engineering

The Engineering Challenge:

The existing cross-sell module was executing expensive, non-indexed SQL joins on every page load, causing high TTFB (Time to First Byte). On the frontend, heavy legacy JS libraries were blocking the main thread, killing user engagement.

My Solution:

I rewrote the backend logic in PHP to optimize MySQL query paths and implemented Redis key-value caching (15-min TTL) to offload the database. For the UI, I architected a lightweight, lazy-loaded slider using Alpine.js to eliminate layout shifts.

The Tech Stack:

PHP 8.1 (Strict) • MySQL Optimization • Redis • Alpine.js • Magento Modules

Generated ₪1,000,000 in incremental monthly revenue by maximizing cross-sell visibility and load speed.
